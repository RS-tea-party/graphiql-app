## Файловая структура проекта

```
graphql-app
│
│   README.md
│   package.json
│   ...
│   # В корневом каталоге находятся все файлы и папки настроек
│   # проекта(prettier, test, eslint, git etc.) и index.html
│
└───src
│   │
│   │   main.tsx  # В корне ./src находятся root-элемент main.tsx(с настройкой роутера)
│   │   global.css/scss  # и файл глобальных стилей
│   │
│   └───__tests__  # Каталог с файлами теста с названиями соотв. компонентов
│   │
│   └───assets  # Каталог с картинками, шрифтами, иконками в соответствующих папках
│   │
│   └───components   # Каталог с компонентами проекта
│   │   │
│   │   │   Component1  # Каждый компонент в отдельной папке, внутри которой
│   │   │   Component2  # tsx и файл стилей, если нет возможности описать их inline
│   │   │   ...
│   │   │
│   │   └───_ui  # Каталог с компонентами ui, каждый в отдельной папке
│   │
│   └───dto  # Каталог для частей проекта
│   │   │
│   │   │   constants.ts  # Все константы проекта
│   │   │   types.ts  # Вся типизация проекта( + интерфейсы )
│   │
│   └───helpers  # Каталог для вспомогательных функций проекта(кастомных, схем, фабрик)
│   │
│   └───hooks  # Каталог с кастомными hooks
│   │
│   └───layouts  # Каталог с общими шаблонами частей страниц
│   │
│   └───service  # Каталог c необходимымы файлами для работы в внешней api (GraphQL)
│   │
│   └───store  # Каталог для всех файлов состояния приложения (RTK, reducers ..)
│   │   └───slices  # Каталог для всех slices
│   │   └───middlewares  # Каталог для всех middlewares
│
└───docs
    └───File.md  # Файлы документации
```
